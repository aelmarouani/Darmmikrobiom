Schritt 3 · Detail 1 von 3

Prevalence-Filter: Rauschen entfernen

Warum filtern?
Ein MSP der nur in 2 von 6.014 Proben vorkommt liefert dem Modell kein stabiles Signal — nur Rauschen.

Schwellenwert: MSP muss in mindestens 10% der Proben (= 601 Proben) detektiert werden.

Ergebnis: 1.990 → ~553 MSPs — 72% der Features entfernt, aber kaum Informationsverlust.
Prävalenz-Verteilung:
~553 MSPs in ≥10% der Proben → behalten
~1.437 MSPs in <10% der Proben → entfernt
Code (Notebook – Abschnitt 4):
# Wie oft kommt jedes MSP vor?
prevalence = (X > 0).mean(axis=0)

# Filter: min. 10% Prävalenz
keep_mask = prevalence >= 0.10

X_filtered = X[:, keep_mask]

# Ergebnis:
print(X.shape) # (6014, 1990)
print(X_filtered.shape) # (6014, ~553)
⚠ Wichtige Erkenntnis später:
XGBoost profitiert nicht vom Filter — das Modell kann intern mit spärlichen Features umgehen. Dieser Schritt hilft anderen Modellen (RF, Extra Trees), XGBoost aber kaum.

Schritt 3 · Detail 2 von 3

log1p-Transformation: Schiefe korrigieren

Das Problem: Extreme Rechtsschieflage
Mikrobiom-Abundanzen sind extrem ungleich verteilt:
Ein dominantes Bakterium kann 30% aller Abundanzen ausmachen, während tausende Arten bei ~0.0001% liegen.

Ohne Transformation lernt das Modell hauptsächlich von den dominanten Arten — schwächere Signale gehen verloren.
Beispielwerte:

import numpy as np

# log(1 + x)
# log(0) = undefiniert → log1p löst das
X_log = np.log1p(X_filtered)

# Alternativ mit Skalierung:
X_log = np.log1p(X_filtered * 1e6)

# Ergebnis: gleichmäßigere Verteilung
# Modell kann besser lernen
✅ log1p behalten wir!
Einziger Schritt der XGBoost wirklich hilft — beschleunigt Training & stabilisiert numerisch.

Diversity-Features: Biologisches Vorwissen hinzufügen

Warum Diversity-Features?
Erkrankte haben nachweislich weniger Artenvielfalt im Darm (Dysbiose) — das ist in der Literatur gut belegt.

Statt das Modell das selbst entdecken zu lassen, fügen wir diese biologische Information explizit als Features hinzu.
FeatureFormelBei Dysbiose
Shannon-Index−∑(p·log p)↓ niedriger
Simpson-Index∑(p²)↑ höher
Observed MGSAnzahl > 0↓ geringer
Pielou EvennessH / log(S)↓ ungleichmäßig
Code (Notebook – Abschnitt 5):
# Relative Abundanzen (Zeile → 1.0)
p = X / X.sum(axis=1, keepdims=True)

# Shannon: H = -sum(p * log(p))
p_safe = np.where(p>0, p, 1)
shannon = -(p * np.log(p_safe)).sum(axis=1)

# Simpson: D = sum(p²)
simpson = (p**2).sum(axis=1)

# Observed: Anzahl MSPs > 0
observed = (X > 0).sum(axis=1)

# Evenness: J = H / log(S)
evenness = shannon / np.log(observed)
